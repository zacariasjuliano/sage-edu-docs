📘 Development Process

SAGE Education follows a structured development cycle, ensuring quality, security and stability with each new version. Stable releases are made available annually (every 12 months). Among these releases, the following are distributed:

  • Quarterly updates (every 3 months), bringing improvements, optimizations and small features.

  • Monthly security updates, which ensure the integrity, protection, and usability of the system, are released on the 12th day.

🔄 Launch Process Steps

The process of releasing a new stable version is carefully coordinated:

    1. Feature Freeze Approximately two months before the release of the stable version, new features are no longer added. The focus becomes stabilization, testing and bug fixes.
    1. Code Freeze About a month before release, the code freeze period enters, in which only critical fixes are allowed. At this stage, the release candidate (RC) is generated.
    1. Update of Environments and Communication to Teams With the RC available, the demo community server is updated, allowing users and partners to validate the new version. Documentation, support and integration teams are also notified to begin their adaptation processes.
    1. Stable Version Release After validation, final tests and necessary adjustments, the stable version is officially available.
